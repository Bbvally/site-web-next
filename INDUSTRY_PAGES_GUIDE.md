## Template Structure

Each industry page should follow this structure:

```
1. Hero Section
   - Industry-specific headline
   - Industry-specific pain point
   - CTA button

2. Problem & Solution
   - What's the challenge for this industry
   - How reviews solve it
   - Key statistics

3. Benefits Section (6 boxes)
   - Industry-specific benefits
   - Real ROI metrics
   - Competitive advantage

4. How It Works
   - 4-step process (customizable)
   - Industry-specific examples

5. Pricing
   - 3 tiers (Starter, Professional, Premium)
   - Industry-specific pricing

6. FAQ
   - Industry-specific questions
   - Address common concerns

7. CTA Section
   - Final push to action
   - Lead magnet (free reviews)
```

## Implementation Steps

1. **Create all pages** using the restaurant/hotel templates as examples
2. **Customize content** for each industry
3. **Add industry-specific CTAs** (booking, inquiry forms, etc.)
4. **Optimize for keywords** like "Law firm reviews Nigeria", "Hospital reviews Lagos", etc.
5. **Add schema.org markup** for service pages
6. **Link from main industries page** to all sub-pages
7. **Update sitemap** to include all new pages

## SEO Strategy for Industry Pages

### Keywords to Target
Each industry should target:
- `[Industry] Google reviews [city]`
- `Best [industry] in [city]`
- `Increase [industry] customers reviews`
- `[Industry] review collection service Nigeria`

### Optimization
- Meta title: 60 chars max
- Meta description: 155 chars max
- H1: Industry-specific, keyword-rich
- Internal links: Link to other industries
- External links: Link to relevant resources
- Schema markup: Service + Organization + LocalBusiness

## Alternative: Dynamic Pages

Instead of creating 16+ separate files, you could create:
1. A single dynamic route: `[locale]/industries/[slug]/page.tsx`
2. Industry data file: `src/data/industries.json`
3. Single component that renders any industry

This would reduce code duplication but require more setup time.
